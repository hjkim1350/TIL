문자열은 immutable(변경 불가능한) 자료형

1) 문자열 슬라이싱
- 문자열을 조작하는 것 중 하나일 뿐.

    a   b   c   d   e
i   0   1   2   3   4
i  -5  -4  -3  -2  -1

마이너스 인덱스가 헷갈리면, 마이너스 인덱스값 앞에 문자열 길이를 두고 계산하면 바로 양의 정수 인덱스값이 나옴.


2) 문자열 메서드
2-1) .split(기준문자)
복잡도 왠지 O(1)일것 같은뎅.

2-2) .strip(제거할 문자)
문자열의 양쪽 끝에 있는 특정 문자를 모두 제거한 새로운 문자열 반환
괄호안에 아무것도 넣지 않으면 자동으로 공백을 제거문자로 설정
제거할 문자를 여러개 넣으면 해당하는 모든 문자들을 제거
* 원본 문자열은 변하지 않음

2-3) .find(찾는 문자)
특정 문자가 처음으로 나타나는 위치(인덱스)를 반환. 찾는 문자가 없다면 -1을 반환
https://www.acmicpc.net/problem/10988

2-4) .index(찾는 문자)
특정 문자가 처음으로 나타나는 위치(인덱스)를 반환. 찾는 문자가 없다면 오류 발생
오류 자체가 여기서 동작을 멈춰야할 때를 알려주는 역할이기도 함

2-5) .count(개수를 셀 문자)
문자열에서 특정 문자가 몇 개인지 반환. 문자 뿐만 아니라, 문자열의 개수도 확인 가능

2-6) .replace(기존문자, 새로운 문자)
문자열에서 기존 문자를 새로운 문자로 수정한 새로운 문자열 반환.
특정 문자를 빈 문자열("")로 수정하여 마치 해당 문자를 삭제한 것과 같은 효과 가능

2-7) 삽입할 문자.join(iterable)
https://www.acmicpc.net/problem/17249

문자를 저장하는 형태를 표준화. 각 문자를 표현하는데 8비트.
